Overview

Logstash is a data processing pipeline that ingests data from multiple sources, transforms it, and sends it to your preferred destination. Commonly used with Elasticsearch and Kibana.

Quick Start

  1. Access Logstash logs/metrics via Elasticsearch and Kibana
  2. Configure input, filter, and output plugins in logstash.conf
  3. Start sending data from Filebeat, Beats, or custom sources

Configuration

Setting Example Description
input beats { port => 5044 } Define data sources such as Beats, HTTP, or TCP
filter grok { match => { "message" => "%{COMBINEDAPACHELOG}" } } Transform data with grok, mutate, date, etc.
output elasticsearch { hosts => ["http://elasticsearch:9200"] } Send data to Elasticsearch or other destinations
